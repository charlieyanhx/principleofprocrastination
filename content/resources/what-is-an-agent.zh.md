AI Agent 就是一个能感知环境、做出决策、采取行动的程序。就这么简单。其余的一切都是工程细节。

这个定义已经稳定了三十多年。1995 年，Stuart Russell 和 Peter Norvig 在《Artificial Intelligence: A Modern Approach》第一版中写下了这个定义，到 2020 年出版的第四版依然成立。Agent 通过传感器感知环境，通过内部逻辑做出决策，通过执行器对环境产生作用。不管这个 Agent 是恒温器还是调度十几个 API 调用的大语言模型，底层结构是一样的。

最近围绕"AI Agent"的兴奋，并不是因为什么新概念，而是因为一个老概念变得空前强大了。要理解发生了什么变化以及这意味着什么，最好从最基础的地方开始。

## 你已经认识的最简单的 Agent

恒温器就是一个 Agent。它通过温度传感器感知室温，有一条决策规则：如果温度低于设定值，就打开暖气。然后它通过开关暖炉来执行这个决策。

这不是比喻。按照计算机科学的正式定义，恒温器完全符合 Agent 的标准 -- 有传感器、有决策函数、有执行器。感知、决策、行动。

当然，日常对话中没人会把恒温器叫做"AI Agent"。因为它的决策逻辑太简单了 -- 就一条 if-then 规则。但它的架构和今天部署的最复杂的 Agent 系统是完全一样的。区别仅在于中间那层 -- 感知和行动之间的决策层 -- 装的是什么。

## 决策能力的四个层级

Russell 和 Norvig 描述了一个 Agent 架构的光谱，按照决策逻辑的复杂度由低到高排列。

**简单反射型 Agent** 只对当前感知到的信息做直接响应。恒温器是一个例子。根据关键词拦截邮件的垃圾过滤器也是。没有记忆，没有规划，只有"刺激-反应"。

**模型型 Agent** 维护着一个对世界的内部模型。它有记忆。扫地机器人会构建你家户型的地图，不是撞到障碍物才反应，而是利用地图来规划下一步去哪里。

**目标型 Agent** 有明确的目标。它不只是反应或建模，它会规划。GPS 导航就是目标型 Agent：给定目的地和路况地图，它搜索一条能到达目的地的路线，能评估多条可能的路径并选出最优解。

**效用型 Agent** 能权衡利弊。自动驾驶汽车不只是找到一条路线，它需要在行驶时间、油耗、乘坐舒适度和安全性之间做平衡。它有一个效用函数，能对不同结果排序，选出期望值最高的方案。

这四个层级不是刚性的分类，而是一个连续光谱上的标记点。但它们揭示了一个关键洞察：让 Agent 变得更强大，核心是给它更丰富的决策方式。更好的感知有帮助，更多的行动能力有帮助，但真正的杠杆在决策层。

## 大语言模型如何改变了一切

几十年来，"感知-决策-行动"循环中最难的部分就是"决策"。构建决策逻辑意味着手工编写规则、训练专用模型，或者费力地定义奖励函数。每种方法都在狭窄的领域里管用，但一旦任务需要通用推理或自然语言理解，就会碰壁。

大语言模型改变了这个局面。GPT-4、Claude 等模型在 Agent 决策所需的推理能力上表现出乎意料地好：把目标拆解成步骤、评估选项、理解模糊指令、在计划不如预期时做出调整。

这不是大语言模型最初被设计来做的事。它们的训练目标是预测文本序列中的下一个 token。但在数万亿字的人类文本上训练的结果，是产生了某种可以充当通用推理引擎的东西。

引爆当前 AI Agent 浪潮的核心洞察是：拿一个大语言模型，给它感知能力（读取外部数据），给它行动能力（调用工具和 API），让它用基于语言的推理来做决策。于是你就得到了一个目标型甚至效用型的 Agent，能在广泛的任务上工作，而不需要为每个任务单独编写决策逻辑。

## LLM Agent 的四个组件

一个现代 LLM Agent 通常由四个部分组成。

**语言模型** 是核心，充当推理引擎。"决策"就发生在这里。模型接收对当前状况、目标和可用操作的描述，然后生成计划或下一步动作。

**工具** 让 Agent 能对外部世界产生作用。可能是 API 调用（发邮件、查数据库、下订单），代码执行环境，浏览器，或者与其他软件系统的连接。没有工具，LLM 只能生成文本。有了工具，它就能做事。

**记忆** 在步骤之间持续存在。短期记忆（对话上下文）让 Agent 能追踪已经做了什么。长期记忆（存在数据库或文件系统里）让它能回忆之前的信息，或者引用无法放进单次提示词的大量知识。

**控制循环** 把一切串联起来。Agent 感知（读取新信息）、决策（调用 LLM 推理下一步该做什么）、行动（调用工具），然后再次感知（读取结果）。这个循环持续运行，直到任务完成、Agent 卡住、或者人工介入。

如果这听起来像是"把 LLM 塞进一个软件程序的中间" -- 没错，就是这样。没有哪个单独的组件是神奇的。关键在于 LLM 足够灵活，能在无需为每个任务显式编程的情况下处理各种决策。

## 从一个 Agent 到多个

一旦你有了能推理和行动的 Agent，自然会产生一个问题：如果让好几个 Agent 一起工作会怎样？

多智能体系统并不新鲜。计算机科学家从 1980 年代就开始研究了。思路很直接：与其让一个 Agent 什么都干，不如让多个 Agent 各自负责一个角色或专长，协作完成任务。

在大语言模型时代，这表现为一组专业化的 Agent 由一个编排器协调。一个 Agent 可能负责调研 -- 搜索网络、整理信息。另一个负责数据分析 -- 编写并运行代码。第三个负责撰写报告。编排器决定调用哪个 Agent，在它们之间传递上下文，最后组装输出结果。

微软的 AutoGen、开源项目 CrewAI 等框架让搭建这类系统变得相对简单。编排器本身通常也是一个 LLM，利用推理能力把任务路由给合适的专家。

但多智能体系统有实实在在的代价。2025 年的研究发现，结构不良的多智能体网络会把错误放大 17 倍，远高于单 Agent 基线。更多的 Agent 意味着更多的协调开销，更多的沟通失误可能性，以及更多默默出错的地方。对 AI 团队来说，协调税同样存在。

实践中的教训：当任务确实需要不同的能力或视角时，多智能体系统很强大。当一个配置良好的单 Agent 就够用时，多智能体就是浪费。什么时候该把一个 Agent 拆成多个，是工程判断，不是默认选择。

## 今天 Agent 能做什么，不能做什么

截至 2026 年初，LLM 驱动的 Agent 已经在一系列场景中投入生产。客服 Agent 处理多步骤的支持请求。编码 Agent 编写、测试和调试软件。研究 Agent 从数十个来源收集和整合信息。运维 Agent 监控系统并采取纠正措施。

这些 Agent 确实有用。也确实有局限。

它们会产生幻觉 -- 生成自信但错误的信息。在上下文过长时会丢失对长期任务的追踪。在需要精确数值推理或严格逻辑推导的任务上表现吃力。它们能在真实世界中采取行动，这意味着犯的错误也是真实的。一个聊天机器人写了一段糟糕的文字，只是浪费你的时间。一个 Agent 给客户发了封错误的邮件，那是真正的事故。

美国政府问责局（GAO）在 2025 年专门发布了一份关于 AI Agent 的技术评估，正是因为这些系统正在进入医疗、金融、政府运作等高风险领域。报告指出，Agent 之所以引入新的风险类别，恰恰因为它们是自主行动的。

这不意味着 Agent 不值得用。而是意味着部署时需要适当的护栏：在关键决策点设置人工审核、限制自主行动的范围、记录和监控 Agent 的行为及其原因、以及当 Agent 遇到超出能力范围的情况时有清晰的升级路径。

## 值得保留的思维模型

理解 AI Agent 最简单也最持久的方式是：

Agent 是一个感知、决策、行动的程序。让现代 AI Agent 区别于教科书上那些 Agent 的，是决策层装了什么：一个能用自然语言推理、使用工具、适应新情况而无需重新编程的大语言模型。

这是一种重要的能力。但它不是魔法。不意味着 Agent 什么都能做。它意味着 Agent 能比以往任何软件架构更灵活地处理更广泛的任务。

炒作周期会过去。"感知-决策-行动"的循环不会变。每当有人推出新的 Agent 框架或宣布 Agent 能力的突破，你都可以问三个问题：它能感知什么？它怎么决策？它能做什么？答案会告诉你所有你需要知道的。
